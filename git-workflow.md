# Git Workflow

## Purpose

Git is used in this portfolio to show a professional workflow for tracking project progress, organizing artifacts, and documenting decisions over time.

## Workflow Used

1. Create a repository with a clear folder structure.
2. Add project files in small, logical groups.
3. Use descriptive commit messages that explain what changed.
4. Keep source data, notebooks, documentation, and reflections organized by project.
5. Push the repository to GitHub so the work can be reviewed externally.

## Example Commit Plan

| Commit | Purpose |
| --- | --- |
| `Initialize AI product portfolio structure` | Create repository folders and top-level documentation |
| `Add FoodHub EDA dataset and notebook` | Add technical analysis materials |
| `Add AI prompt engineering artifacts` | Document prompt strategy and before/after examples |
| `Add AI business use case documentation` | Add product workflow and ethics review |
| `Add reflections and presentation materials` | Complete the portfolio narrative |

## Git Skills Demonstrated

- Repository initialization
- File staging and commits
- Organized project directories
- Markdown documentation
- GitHub remote publishing
- Versioned portfolio artifacts

## Why This Matters

Git creates a transparent history of the work. For a product analyst or AI-enabled analyst, this matters because it shows process discipline, reproducibility, and the ability to collaborate in a technical environment.

# Personalization Guide

## Purpose

This guide explains how to turn hobbies into portfolio artifacts without making the portfolio feel random.

## Recommended Format

For each hobby-based widget, document:

1. The hobby or interest
2. The user problem or question
3. The data or inputs used
4. The AI support used
5. The output or decision supported
6. The skill demonstrated

## Example

### Hobby

Trying new restaurants.

### Product Question

How can someone choose a restaurant faster based on cuisine, cost, ratings, and delivery time?

### Widget Idea

A restaurant decision helper that filters options and explains the best match.

### AI Use

Use AI to generate comparison criteria, draft recommendation text, and challenge whether the recommendation is supported by data.

### Skill Demonstrated

Data filtering, recommendation logic, user experience thinking, and business communication.

## Strong Hobby Project Pattern

The strongest hobby widgets should still show a professional skill:

- A cooking widget can show recommendation logic.
- A travel widget can show decision modeling.
- A music widget can show categorization and personalization.
- A fitness widget can show metrics and progress tracking.
- A reading widget can show summarization and knowledge management.

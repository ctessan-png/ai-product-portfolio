# Build Plan Without Building Yet

## Phase 1: Gather Inspiration

Collect 8-12 Pinterest screenshots or saved examples of iPhone widgets that match the desired style.

For each example, note:

- color palette
- typography style
- layout type
- icon style
- mood or feeling
- what information the widget shows

## Phase 2: Define The Style Guide

Create a short style guide before writing code.

Include:

- 3-5 colors
- heading and body font direction
- border radius style
- image treatment
- spacing rules
- icon style
- tone of writing

## Phase 3: Choose Widget Concepts

Pick 3-5 widgets that connect to real interests.

Each concept should answer:

- Who is this for?
- What does it help with?
- What data or input does it use?
- What does AI help create?
- What skill does it show in the portfolio?

## Phase 4: Create Static Mockups

Start with web-based mockups before native iPhone development.

This is easier to review in GitHub and easier to style quickly.

Possible artifacts:

- HTML widget grid
- CSS style system
- sample data file
- screenshots
- reflection document

## Phase 5: Decide On Native iPhone Widgets

After the style and concepts are clear, decide whether to build native widgets with:

- Xcode
- SwiftUI
- WidgetKit
- App Intents for interactivity

Native widgets should come after the portfolio story and mockups are strong.
